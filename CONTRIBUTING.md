# Contributing to Ktebli

Thanks for helping improve Ktebli.

## Getting Started
1. Read the project scope in `docs/mvp-scope.md`.
2. Read the architecture in `docs/architecture.md`.
3. Keep changes aligned with MVP priorities.

## Workflow
1. Create a branch for your change.
2. Keep pull requests focused and small.
3. Reference related issues in your PR.
4. Ensure documentation is updated when behavior changes.

## Issue Reporting
Use the provided issue templates for:
- Bug reports
- Feature requests
- Tasks

## Pull Requests
- Use the PR template.
- Include clear summary and testing notes.
- Keep changes scoped to a single concern.

## Code of Conduct
Be respectful and constructive in all discussions.
